# رحّال AI — مساعدك الذكي للسفر ✈️

تطبيق Flutter ذكي لتخطيط الرحلات باستخدام الذكاء الاصطناعي (Claude AI / Gemini).

## المتطلبات (Prerequisites)
- Flutter SDK >= 3.22
- Node.js >= 18
- حساب مشروع Firebase مع تفعيل (Auth + Firestore)
- مفتاح Anthropic API (أساسي لتوليد الخطط بالذكاء الاصطناعي)

---

## 🚀 Quick Setup Guide (دليل الإعداد السريع)

### 1. إعداد السيرفر الخلفي (Backend Setup - server.js)
1. انتقل إلى مجلد المشروع وقم بتثبيت الحزم:
   ```bash
   npm install
   ```
2. انسخ ملف الإعدادات البيئية:
   ```bash
   cp .env.example .env
   ```
3. املأ المفاتيح الخاصة بك في ملف `.env`:
   - `ANTHROPIC_API_KEY`: مفتاح Claude AI (مطلوب للتوليد الأساسي).
   - `GEMINI_API_KEY`: مفتاح Gemini AI (اختياري للتبديل عند الطوارئ).
   - `GOOGLE_PLACES_API_KEY`: مفتاح Google Places (اختياري للتحقق الفعلي وجلب إحداثيات الأماكن).
   - `UNSPLASH_ACCESS_KEY`: مفتاح جلب صور المزارات السياحية بجودة عالية (اختياري).
   - `PEXELS_API_KEY`: مفتاح جلب صور بديلة (اختياري).
   - `OPENWEATHER_API_KEY`: مفتاح جلب حالة الطقس للوجهات (اختياري).

4. ابدأ تشغيل السيرفر الخلفي (يعمل افتراضياً على المنفذ 3000):
   ```bash
   node server.js
   ```

### 2. إعداد تطبيق الفلاتر (Flutter Setup)
- **محاكي الأندرويد (Android Emulator)**: لا يتطلب أي إعداد إضافي (يكتشف تلقائياً العنوان `http://10.0.2.2:3000`).
- **محاكي الـ iOS (iOS Simulator)**: يعمل تلقائياً باستخدام العنوان المحلي `http://127.0.0.1:3000`.
- **جهاز حقيقي للتجربة (Physical Device)**: قم بتشغيل التطبيق وتمرير عنوان الآي بي الخاص بجهازك المضيف على الشبكة المحلية:
  ```bash
  flutter run --dart-define=PROXY_BASE_URL=http://<YOUR_IP>:3000
  ```
- **وضع الإنتاج والرفع (Production Mode)**: يستخدم التطبيق تلقائياً الرابط السحابي المرفوع على Render، أو يمكنك تمرير رابط خادمك الخاص:
  ```bash
  flutter run --dart-define=PROXY_BASE_URL=https://your-production-server.com
  ```

## 🛠️ Troubleshooting & Common Issues (حلول المشاكل الشائعة)

- 🔴 **مشكلة "BOTTOM OVERFLOWED" (تداخل العناصر أسفل الشاشة)**:
  - تم إصلاحها بالكامل في التحديث الحالي باستخدام Safe Areas و Custom Scroll Views.
- 🔴 **فشل توليد الرحلة (خطة فاشلة)**:
  - تأكد من صحة صلاحية مفتاح `ANTHROPIC_API_KEY` في ملف `.env` وعدم انتهاء رصيد الحساب.
- 🔴 **خطأ "Unhandled Exception"**:
  - تأكد من تشغيل `flutter pub get` لتحديث حزمة `geocoding` وبقية الحزم الفرعية.
- 🟠 **ظهور بيانات وهمية وتجريبية فقط (Mock Data showing)**:
  - يعني هذا تعذر اتصال التطبيق بالسيرفر الخلفي (الـ Proxy). يرجى التأكد من تشغيل `node server.js` وأن الجهاز متصل بنفس شبكة خادم التطوير عند استخدام جهاز حقيقي.
