# رحّال AI — مساعدك الذكي للسفر ✈️

تطبيق Flutter ذكي لتخطيط الرحلات باستخدام الذكاء الاصطناعي (Groq / Gemini).

## المتطلبات (Prerequisites)
- Flutter SDK >= 3.22
- Node.js >= 18
- حساب مشروع Firebase مع تفعيل (Auth + Firestore)
- مفتاح Groq API مجاني (المحرك الأساسي لتوليد الخطط)

---

## 🚀 Quick Setup Guide (دليل الإعداد السريع)

### 1. إعداد السيرفر الخلفي (Backend Setup - server.js)
1. انتقل إلى مجلد المشروع وقم بتثبيت الحزم:
   ```bash
   npm install
   ```
2. انسخ ملف الإعدادات البيئية:
   ```bash
   cp .env.example .env
   ```
3. املأ المفاتيح الخاصة بك في ملف `.env`:
   - `GROQ_API_KEY`: مفتاح Groq (**المحرك الأساسي** — مجاني من https://console.groq.com).
   - `GEMINI_API_KEY`: مفتاح Gemini (احتياطي تلقائي إذا فشل Groq).
   - `GOOGLE_PLACES_API_KEY`: مفتاح Google Places (للتحقق من الأماكن وإحداثياتها، وحل أسماء المدن غير المدرجة في القاموس).
   - `UNSPLASH_ACCESS_KEY`: مفتاح جلب صور المزارات السياحية بجودة عالية (اختياري).
   - `PEXELS_API_KEY`: مفتاح جلب صور بديلة (اختياري).
   - `OPENWEATHER_API_KEY`: مفتاح جلب حالة الطقس للوجهات (اختياري).

4. ابدأ تشغيل السيرفر الخلفي (يعمل افتراضياً على المنفذ 3000):
   ```bash
   node server.js
   ```
   عند الإقلاع يطبع السيرفر أي المفاتيح مُفعّلة وأي محرك ذكاء اصطناعي سيُستخدم.

### 2. إعداد تطبيق الفلاتر (Flutter Setup)
- **الويب (Chrome)**: يتصل تلقائياً بـ `http://localhost:3000`.
- **محاكي الأندرويد (Android Emulator)**: لا يتطلب إعداداً إضافياً (يستخدم `http://10.0.2.2:3000`).
- **محاكي الـ iOS (iOS Simulator)**: يعمل تلقائياً باستخدام `http://127.0.0.1:3000`.
- **جهاز أندرويد حقيقي (Physical Device)**: ⚠️ نسخة الـ debug تستخدم افتراضياً العنوان `10.0.2.2:3000` وهو **خاص بالمحاكي فقط ولا يعمل على جهاز حقيقي**. لذلك مرّر عنوان الخادم صراحةً — إما خادمك المنشور:
  ```bash
  flutter run --dart-define=PROXY_BASE_URL=https://rahhalflutter-production.up.railway.app
  ```
  أو خادمك المحلي عبر عنوان الـ IP لجهازك على نفس الشبكة:
  ```bash
  flutter run --dart-define=PROXY_BASE_URL=http://<YOUR_IP>:3000
  ```
  ونفس الشيء عند بناء ملف APK للتثبيت اليدوي:
  ```bash
  flutter build apk --debug --dart-define=PROXY_BASE_URL=https://rahhalflutter-production.up.railway.app
  ```
- **وضع الإنتاج (Release Mode)**: يستخدم التطبيق تلقائياً الخادم المنشور على **Railway** دون الحاجة لأي إعداد.

### 3. النشر على Railway
الخادم الخلفي منشور على Railway ويُعاد نشره تلقائياً عند الدفع إلى فرع `main`.
> ⚠️ مهم: ملف `.env` **لا يُرفع** إلى Git. يجب إضافة المفاتيح (وأهمها `GROQ_API_KEY`) يدوياً في لوحة Railway ضمن تبويب **Variables**، وإلا فلن يجدها الخادم المنشور ولن تعمل عملية توليد الرحلات على الهاتف.

## 🤖 كيف يعمل محرك الذكاء الاصطناعي

- **المحرك الأساسي**: Groq (نموذج `openai/gpt-oss-20b`) عبر واجهة متوافقة مع OpenAI.
- **الاحتياطي**: Google Gemini — يُستخدم تلقائياً فقط إذا فشل Groq.
- **حل الوجهات العربية**: النماذج لا تقرأ اسم المدينة العربية بثقة داخل الـ prompt الطويل (فتسأل "أي مدينة؟" أو تستبدلها بمدينة أشهر). لذلك تُحلّ الوجهة إلى اسم إنجليزي **قبل** التوليد عبر:
  1. قاموس محلي مجاني (`AR_CITY_DICTIONARY`) يغطي محافظات العراق ومدن المنطقة الكبرى.
  2. Google Places كاحتياطي للأسماء غير المدرجة.

  ثم تُحقن المدينة المُحلّلة كتوجيه صارم "ممنوع الاستبدال"، وتُستخدم أيضاً لرفض أي رحلة تعود لمدينة خاطئة.

## 🛠️ Troubleshooting & Common Issues (حلول المشاكل الشائعة)

- 🔴 **فشل توليد الرحلة على جهاز حقيقي (لا تظهر أي رحلة)**:
  - غالباً لأن نسخة الـ debug تحاول الاتصال بـ `10.0.2.2:3000` (عنوان المحاكي). أعد البناء مع `--dart-define=PROXY_BASE_URL=...` كما في القسم أعلاه.
- 🔴 **فشل توليد الرحلة (خطأ من الخادم)**:
  - تأكد من صحة `GROQ_API_KEY` في `.env` محلياً، **وفي متغيرات Railway** للنسخة المنشورة.
  - الطبقة المجانية في Groq محدودة بـ ~8000 توكن/دقيقة — الطلبات المتتالية السريعة قد تُرفض مؤقتاً (429). انتظر دقيقة وأعد المحاولة.
  - الطبقة المجانية في Gemini محدودة بـ 20 طلب/يوم، وGoogle Places بـ 100 بحث/يوم.
- 🔴 **الرحلة تعود لمدينة خاطئة**:
  - إن كانت الوجهة غير مدرجة في القاموس ونفدت حصة Google Places، قد يفشل حل الوجهة. أضف المدينة إلى `AR_CITY_DICTIONARY` في `server.js`.
- 🔴 **بطء أول طلب (~30 ثانية)**:
  - خادم Railway المجاني يدخل وضع السكون؛ أول طلب يوقظه والطلبات التالية أسرع.
- 🟠 **ظهور بيانات وهمية وتجريبية فقط (Mock Data showing)**:
  - يعني تعذّر اتصال التطبيق بالسيرفر الخلفي. تأكد من تشغيل `node server.js` أو من صحة رابط الخادم الممرّر.

### ملاحظات لبناء APK على أجهزة محدودة الذاكرة
- بناء `--release` قد يفشل بنفاد الذاكرة على أجهزة 8GB؛ استخدم `--debug` مع `--dart-define` لتحديد الخادم.
- أغلق Chrome وأوقف عمليات Gradle القديمة (`cd android && ./gradlew --stop`) قبل البناء.
- عند ظهور خطأ `immutable workspace ... modified` احذف مجلد `~/.gradle/caches/<version>/transforms` وأعد البناء.
- بعض أجهزة Realme/Oppo (ColorOS) تُثبّت التطبيق على مستخدم ضيف متوقف؛ ثبّت صراحةً على المستخدم الرئيسي:
  ```bash
  adb push build/app/outputs/flutter-apk/app-debug.apk /data/local/tmp/rahhal.apk
  adb shell pm install -r --user 0 /data/local/tmp/rahhal.apk
  ```
